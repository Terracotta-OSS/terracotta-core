Terracotta Second-Level Cache for Hibernate Views

If you are using the Terracotta Second-Level Cache for Hibernate with your Hibernate-based application, the Hibernate and Second-Level Cache views offers you the following:

  • Deep visibility into Hibernate and cached data
  • Live statistics
  • Graphs, including puts and misses
  • Historical data for trend analysis
  • Parameters for control over the size and efficiency of cache regions
  • A configuration generator

Note that the settings you change in the Second-Level Cache view are not saved to the cache configuration file and are not persisted beyond the lifetime of the cache.

For more information on testing and tuning the Terracotta Second-Level Cache for Hibernate, go here.

Hibernate View

The Hibernate view displays Hibernate statistics. Some of the main tasks you can perform in this view are:

  • View statistics for the entire cluster on Hibernate entities, collections, and queries.
  • View statistics for each Terracotta client (application server) on Hibernate entities, collections, and queries.

Second-Level Cache View

The Second-Level Cache view provides performance statistics and includes per-region cache configuration. Some of the main tasks you can perform in this view are:

  • View both live and historical performance metrics in graphs.
  • Enable and disable cache regions.
  • Flush cache regions.
  • Set eviction parameters per region.
  • Generate a configuration file based on settings in the Second-Level Cache view.

Viewing Multiple Caches

If your cluster has more than one second-level cache, use the Persistence Unit drop-down menu to choose the cache you want to view.
