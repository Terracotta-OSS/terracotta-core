Welcome to Terracotta Sessions Configurator

Use Configurator to setup Terracotta Sessions to cluster your existing session-based web application.  Configurator provides a testing environment complete with an installation of Apache Tomcat for clustering two instances of a web application.

The basic process is:
  1. Validate Unclustered WebApp
    2. Import your web application into Configurator and verify that it runs correctly on Tomcat or other web containers you have specified.  In case you do not have access to a webapp of your own, Terracotta provides a couple of sample webapps.

  2. Iterate through the Configuration Process
    3. Run your webapp on Terracotta Sessions and identify session data that should be clustered across the servers.  Restart your webapp and continue to identify data to cluster.  Repeat until your webapp is properly configured to run clustered.

  3. Check Clustering and Fail-Over
    4. Use Configurator's real-time monitor to verify clustering of session data and then shut down one of the Tomcat servers to verify session data is properly maintained.
You are now ready to deploy your webapp with Terracotta Sessions and scale-out across a large number of servers.
